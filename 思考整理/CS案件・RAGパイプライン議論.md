# テーマ：CS導入支援AIの「自己改善ループ」構築と現場での実効性確保

## 1. 現状と事実（Fact）
* カスタマーサクセスのハイタッチ導入支援で、案件精査をAIで自動化・高度化したい。
* 現在、スプレッドシート＋Gemini APIを使い、250レコードを4分割して週1回精査している。
* 「小カテゴリ」でリスクとアクションを分類・管理するテーブルを既に使用している。
* 過去のチャレンジでは、RAGがうまく構築できず現場のアクションにつながっていなかった。

## 2. 違和感・悩み・葛藤（Conflict）
* 「RAGが作りきれていない」という技術的な課題感がある。
* AIが提案するアクションが「詰め合わせ」になり、現場担当者が結局何をすべきか分からず動けていない（渋滞・冗長化）。
* アクションが実行されないことで「正解データ」が取れず、自己改善ループが最初から回っていない。
* アクション提案を最適化する仕組みを作っても、それが大元の「リスク-アクションマスタ」自体の改善にどうつながるのか見えない。

## 3. なぜそう思うか？（Why）
* 「ただAIを動かす」ことではなく、**「CSMが納得して動ける現場のアクション」と「チーム全体の知見の底上げ（マスタ進化）」を両立させたい**という強い理想がある。
* 属人的なCSの知見をシステム化したい一方、現場の負担を増やして運用が形骸化することを極度に恐れている（過去の失敗からの学び）。
* 「AIへの命令」と「現実の実行・評価」が分断されると、結局ツールが「ただの報告書作成器」になってしまう懸念がある。

## 4. 一旦の着地点・次のアクション（Next）
* **結論:** プログラムでのアクション結合をやめ、全ての材料（リスク・期日・接触履歴・成功失敗事例）をGeminiに渡し、「今週やるべき1つのアクション（One Big Action）」を統合生成させる。
* **次の小さな一歩:**
    1.  **カテゴリ整理:** 現在の50個の小カテゴリを「根本原因ベース」で20個程度に統合する。
    2.  **入力の簡素化:** スプレッドシートに「実行チェック」「解決チェック」「理由メモ」の列を追加し、現場の負担を極小化する。
    3.  **ループの設計:** 「現場の実行結果」を「過去事例データベース（RAG用）」として自動蓄積させ、月1回、Geminiにその履歴から「マスタ改善案」を出力させるマクロ改善ループを準備する。
